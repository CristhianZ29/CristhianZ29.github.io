Historia de los Sistemas Operativos

Los sistemas operativos surgieron para facilitar la interacción entre el usuario y el hardware del computador. Antes de su existencia los programas se ejecutaban directamente en la máquina, lo que hacía el proceso complejo y poco eficiente.

  • Década de 1940: Los primeros computadores electrónicos no tenían sistemas operativos. Los programas se cargaban manualmente mediante tarjetas perforadas.
  • Década de 1950: Aparecen los sistemas Batch que permitían ejecutar trabajos por lotes sin intervención del usuario.
  • 1964: IBM desarrolla OS/360, uno de los primeros sistemas operativos capaces de gestionar múltiples tareas.
  • 1969: Se desarrolla UNIX en Bell Labs por Ken Thompson y Dennis Ritchie, introduciendo conceptos clave como portabilidad y multitarea.
  • 1981: Microsoft lanza MS-DOS, ampliamente usado en computadores personales.
  • 1991: Linus Torvalds desarrolla el kernel de Linux, base de muchas distribuciones actuales.
  • Actualidad: Los sistemas operativos modernos gestionan memoria, procesos, almacenamiento, seguridad y permiten ejecutar múltiples aplicaciones simultáneamente.

Algoritmos de Planificación de Procesos

La planificación de procesos es el mecanismo mediante el cual el sistema operativo decide qué proceso utilizará el CPU en cada momento.

  • FCFS (First Come First Served): ejecuta procesos en el orden en que llegan.
  • SJF (Shortest Job First): prioriza los procesos con menor tiempo de ejecución.
  • Round Robin: asigna un intervalo de tiempo a cada proceso para evitar monopolizar el CPU.
  • Prioridad: el sistema ejecuta primero los procesos más importantes.

Estos algoritmos influyen directamente en el rendimiento del sistema, el tiempo de respuesta y la eficiencia en el uso del procesador.

Configuración básica de Debian desde terminal

Debian es una distribución de Linux conocida por su estabilidad. Después de instalar el sistema base, muchas tareas de administración se realizan mediante la terminal utilizando el gestor de paquetes APT.

Actualizar el sistema:

sudo apt update
sudo apt upgrade

Instalar paquetes básicos:

sudo apt install sudo
sudo apt install net-tools
sudo apt install curl

Comandos básicos de información del sistema:

uname -a
whoami
pwd

Estos comandos permiten actualizar el sistema, instalar herramientas necesarias y consultar información básica del entorno.

Ciclo de vida de los procesos

Un proceso es un programa en ejecución. Durante su funcionamiento pasa por distintos estados dentro del sistema operativo.

  • Nuevo: el proceso acaba de crearse.
  • Listo: está preparado para ejecutarse.
  • En ejecución: está usando el CPU.
  • En espera: espera recursos o eventos.
  • Finalizado: el proceso termina su ejecución.

El sistema operativo toma decisiones constantemente para mover procesos entre estos estados y así mantener el sistema funcionando de forma eficiente.
